# 学习心得

## 刷题建议

### 1. 制定计划
- 每天坚持刷 1-3 题
- 先做 Easy，再做 Medium，最后做 Hard
- 按类型刷题，如先刷数组相关的所有题

### 2. 解题步骤
1. **理解题意**：仔细阅读题目，确保理解正确
2. **分析思路**：思考可能的解法，评估复杂度
3. **编写代码**：实现解法，注意细节
4. **测试验证**：用测试用例验证正确性
5. **优化改进**：寻找更优的解法
6. **总结回顾**：记录思路和关键点

### 3. 两语言对比
- **Python**：快速原型开发，代码简洁
- **C++**：性能优化，面试常用
- 两种语言各有优势，都要掌握

### 4. 常见错误
- ✗ 只看答案，不思考
- ✗ 一道题做完就忘
- ✗ 忽视复杂度分析
- ✗ 不测试边界情况

### 5. 正确做法
- ✓ 先独立思考，再参考答案
- ✓ 定期回顾，巩固记忆
- ✓ 每道题都分析复杂度
- ✓ 充分测试，包括边界情况

## 面试准备

### 技术栈优先级
1. **必须掌握**
   - 数组、链表、树、图
   - 排序、查找、递归
   - 动态规划

2. **重要但次要**
   - 栈、队列、哈希表
   - 双指针、滑动窗口
   - 贪心、分治

3. **加分项**
   - 字典树、线段树、并查集
   - 高级图算法
   - 复杂的 DP 优化

### 面试技巧
- 先说思路，再写代码
- 分析时空复杂度
- 考虑边界情况
- 代码可读性很重要
