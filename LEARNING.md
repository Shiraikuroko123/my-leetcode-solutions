# 分类学习路径

AlgoNote 的标签不仅用于筛选，也用于组织复习顺序。建议每一类都按“理解模板、独立实现、构造反例、间隔复习”完成闭环。

## 推荐顺序

1. 数组与哈希：索引、计数、前缀和、快速查找
2. 双指针：有序数组、原地修改、快慢指针
3. 滑动窗口：连续区间、不变量、计数窗口
4. 栈与单调栈：括号、最近关系、下一个更大元素
5. 二分查找：精确值、左右边界、答案空间
6. 链表：哨兵节点、反转、环与快慢指针
7. 树与递归：DFS、BFS、分治、递归返回值
8. 图与搜索：连通分量、拓扑排序、最短路径、并查集
9. 动态规划：状态、转移、初始化、遍历顺序、空间压缩
10. 贪心：局部选择、交换论证、反例验证

## 单题工作流

1. 在官方题面核对输入、输出、约束和函数签名。
2. 写出最直接的暴力解法，并估算时间与空间复杂度。
3. 找出重复计算、可维护的不变量或可利用的单调性。
4. 在编辑器中实现，先覆盖空值、单元素、重复值和极端范围。
5. 运行可见测试，再主动构造一个能击穿错误思路的用例。
6. 查看提示和题解，对比状态定义、边界处理与复杂度。
7. 用另一种语言重写，避免只记住语法片段。
8. 标记完成，并在 1 天、7 天、30 天后重新实现。

## Python 与 C++ 对照

| 关注点 | Python | C++ |
| --- | --- | --- |
| 哈希结构 | `dict` / `set` | `unordered_map` / `unordered_set` |
| 队列 | `collections.deque` | `queue` / `deque` |
| 堆 | `heapq`，默认小根堆 | `priority_queue`，默认大根堆 |
| 整数范围 | 通常无需手动溢出检查 | 注意 `int`、`long long` 与中点计算 |
| 递归 | 注意递归深度 | 注意栈深度和引用传递 |

## 复盘记录

每道题至少记录以下内容：

- 一句话核心观察
- 暴力解法为什么不够
- 优化解法保持了什么不变量
- 一个最容易遗漏的边界
- 时间和空间复杂度
- 下次看到什么信号时应联想到该方法

不要把“看懂答案”记为完成。能够关闭题解、独立写出并解释正确性，才算掌握。
